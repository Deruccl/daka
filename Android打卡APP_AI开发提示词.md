## 1\. 项目总览与核心定位

### 1\.1 项目名称

**「时光印记」\- 全能单机打卡助手**
英文名称：TimeMark \- All\-in\-One Offline Tracker

### 1\.2 产品 Slogan

> 每一次打卡，都是对自己的温柔承诺；每一天记录，都是时光留下的珍贵印记。
> 
> 

### 1\.3 核心价值主张

这是一款**完全离线运行**的 Android 原生打卡应用，用户无需注册账号、无需联网、无需担心数据隐私泄露。所有数据 100% 存储在用户设备本地，真正做到「我的数据我做主」。

应用主打**高度自定义**的打卡体验，用户可以创建任意类型的打卡项目 —— 饮食记录、饮水提醒、运动健身、睡眠追踪、学习计划、读书打卡、冥想练习、护肤打卡、服药提醒等等，一切由用户自己定义。

同时，应用内置**强大的 AI 分析能力**，通过接入用户自己的 AI 模型 API，实现智能识别、数据分析、个性化建议等高级功能。支持多模态与非多模态模型协同工作，在保证功能强大的同时最大化节省 Token 消耗。

### 1\.4 目标用户画像

#### 1\.4\.1 核心用户群

- **年龄**：18\-45 岁

- **特征**：注重自我管理、追求生活品质、有记录习惯、关注数据隐私

- **场景**：健身减脂、健康管理、学习提升、习惯养成、生活记录

#### 1\.4\.2 用户细分

1. **健康管理型用户**：关注饮食热量、饮水量、运动时长、睡眠质量

2. **学习提升型用户**：需要追踪学习时长、阅读页数、单词背诵进度

3. **习惯养成型用户**：希望通过打卡培养早起、冥想、护肤等好习惯

4. **数据极客型用户**：喜欢量化自我、热衷于数据分析和可视化

5. **隐私敏感型用户**：对数据安全有极高要求，拒绝云端存储

### 1\.5 设计哲学

#### 1\.5\.1 极简而不简单

界面设计遵循极简主义美学，但功能上绝不妥协。每一个交互都经过精心设计，让用户在使用中感受到「恰到好处」的舒适感。

#### 1\.5\.2 液态质感

借鉴 iOS 26 的液态玻璃设计语言，但用 Android 原生技术实现。所有卡片、弹窗、导航栏都呈现出半透明、模糊、折射的液态质感，仿佛玻璃上凝结的水珠，晶莹剔透。

#### 1\.5\.3 丝滑流畅

所有动画都遵循物理运动规律，速度曲线自然流畅。页面切换、元素展开、手势交互都要达到 60fps 以上的流畅度，给用户「指哪打哪」的跟手体验。

#### 1\.5\.4 智能但不打扰

AI 功能强大但克制，只在用户需要时提供建议，绝不主动推送、绝不打扰用户。所有分析结果都以优雅的方式呈现，让用户自己决定是否采纳。

#### 1\.5\.5 完全可控

用户对自己的数据有绝对控制权。可以随时导出、随时删除、随时重置。应用不会收集任何用户数据，不会有任何广告，不会有任何强制更新。

### 1\.6 产品愿景

成为 Android 平台上**最好用的单机打卡应用**，让每一个热爱生活、追求进步的用户都能拥有一款纯粹、优雅、强大的自我管理工具。

## 2\. 技术架构与开发规范

### 2\.1 技术栈要求

#### 2\.1\.1 开发语言

- **主语言**：Kotlin（100% Kotlin 代码，禁止使用 Java）

- **协程**：Kotlin Coroutines \+ Flow 处理异步操作

- **序列化**：Kotlinx Serialization

#### 2\.1\.2 UI 框架

- **主框架**：Jetpack Compose（完全使用 Compose 构建 UI，禁止使用 XML 布局）

- **Compose 版本**：使用最新稳定版

- **Material Design**：Material 3（Material You）设计系统

- **动态颜色**：支持 Android 12\+ 的动态取色功能

#### 2\.1\.3 架构模式

- **整体架构**：Clean Architecture \+ MVVM

- **分层结构**：

    - Presentation Layer（UI 层）：Compose \+ ViewModel

    - Domain Layer（领域层）：Use Cases \+ Repository Interfaces

    - Data Layer（数据层）：Repository Impl \+ Data Sources

- **状态管理**：ViewModel \+ StateFlow/SharedFlow

- **依赖注入**：Hilt（推荐）或 Koin（轻量级备选）

#### 2\.1\.4 本地存储

- **结构化数据**：Room Database（SQLite）

- **键值对**：DataStore（Preferences DataStore \+ Proto DataStore）

- **文件存储**：内部存储 \+ 外部存储（用户可选择）

- **图片存储**：App 内部目录 \+ Glide/Coil 图片加载

#### 2\.1\.5 网络请求

- **HTTP 客户端**：OkHttp \+ Retrofit

- **JSON 解析**：Moshi 或 Kotlinx Serialization

- **图片加载**：Coil（Compose 友好）

#### 2\.1\.6 其他核心库

- **日期时间**：java\.time（ThreeTen Abp 兼容低版本）

- **图表**：MPAndroidChart 或 Compose 自定义图表

- **权限**：Accompanist Permissions

- **导航**：Navigation Compose

- **生命周期**：AndroidX Lifecycle

### 2\.2 单机运行架构设计

#### 2\.2\.1 完全离线优先

- 所有核心功能 100% 离线可用

- AI 功能为可选增强，不影响基础功能使用

- 无账号系统、无云端同步、无后台服务

#### 2\.2\.2 数据本地化

- 所有用户数据存储在应用私有目录

- 数据库加密（可选，用户可开启）

- 支持导出为 JSON/CSV 格式

- 支持从备份文件恢复数据

#### 2\.2\.3 AI 功能可选

- AI 分析功能需要用户自行配置 API Key

- 未配置时，应用完全正常运行，只是缺少 AI 分析功能

- 所有 AI 请求都由用户设备直接发起，不经过第三方服务器

### 2\.3 版本兼容要求

- **最低 SDK 版本**：API 26（Android 8\.0 Oreo）

- **目标 SDK 版本**：最新稳定版（当前 API 34）

- **编译 SDK 版本**：最新稳定版

- **设备适配**：手机优先，兼顾平板适配

### 2\.4 代码规范与命名约定

#### 2\.4\.1 包名结构

```Plain Text
com.timemark.app
├── ui              # UI 层
│   ├── theme       # 主题配置
│   ├── screens     # 页面
│   ├── components  # 可复用组件
│   ├── navigation  # 导航配置
│   └── viewmodels  # ViewModel
├── domain          # 领域层
│   ├── model       # 数据模型
│   ├── repository  # 仓库接口
│   └── usecase     # 用例
├── data            # 数据层
│   ├── repository  # 仓库实现
│   ├── local       # 本地数据源
│   │   ├── db      # 数据库
│   │   └── datastore
│   └── remote      # 远程数据源（AI API）
├── di              # 依赖注入
├── utils           # 工具类
└── App.kt          # Application 类
```

#### 2\.4\.2 命名规范

- 类名：PascalCase（大驼峰）

- 函数 / 变量：camelCase（小驼峰）

- 常量：UPPER\_SNAKE\_CASE

- Compose 组件：PascalCase

- 资源文件：snake\_case

#### 2\.4\.3 代码质量要求

- 每个文件不超过 500 行

- 每个函数不超过 50 行

- 关键逻辑必须有注释

- 遵循 Kotlin 官方编码规范

- 避免内存泄漏

### 2\.5 模块化设计

应用采用模块化架构，便于维护和扩展：

1. **app 模块**：主应用模块，包含 UI 和应用入口

2. **core 模块**：核心工具类、扩展函数、通用组件

3. **data 模块**：数据层实现

4. **domain 模块**：领域层逻辑

5. **ai 模块**：AI 相关功能封装

6. **feature 模块**：按功能划分的特性模块

    - feature\-home

    - feature\-tracker

    - feature\-stats

    - feature\-settings

    - feature\-ai

## 4\. 液态玻璃（Liquid Glass）效果实现规范

### 4\.1 效果概述

液态玻璃效果是本应用最核心的视觉特色。它借鉴了 iOS 26 的液态玻璃设计语言，但完全使用 Android 原生技术实现。效果表现为：

- **半透明**：玻璃材质可以看到背后的内容

- **模糊**：背后的内容被高斯模糊，产生景深效果

- **折射**：边缘有微妙的折射变形，模拟真实玻璃

- **高光**：顶部有柔和的高光，模拟光线反射

- **边框**：有细细的高光边框，增强玻璃质感

- **阴影**：柔和的阴影，增加悬浮感

### 4\.2 技术实现方案

#### 4\.2\.1 模糊效果实现

- **主要方案**：使用 RenderScript 或 Android 12\+ 的模糊 API

- **兼容方案**：对于不支持硬件模糊的设备，使用半透明纯色替代

- **模糊半径**：

    - 轻度模糊：10dp（背景层）

    - 中度模糊：20dp（标准卡片）

    - 重度模糊：30dp（弹窗、底部导航）

#### 4\.2\.2 Compose 实现方式

```kotlin
// 玻璃卡片组件示例
@Composable
fun GlassCard(
    modifier: Modifier = Modifier,
    blurRadius: Dp = 20.dp,
    content: @Composable () -> Unit
) {
    // 1. 背景模糊层
    // 2. 半透明渐变叠加层
    // 3. 高光边框层
    // 4. 内容层
}
```

### 4\.3 玻璃材质分级

根据不同的使用场景，玻璃材质分为三个等级：

#### 4\.3\.1 轻薄玻璃（Light Glass）

- **透明度**：70% 不透明度

- **模糊半径**：10dp

- **适用场景**：列表项、小卡片、按钮背景

- **视觉感受**：轻盈、通透

#### 4\.3\.2 标准玻璃（Standard Glass）

- **透明度**：80% 不透明度

- **模糊半径**：20dp

- **适用场景**：主要卡片、内容容器

- **视觉感受**：平衡、舒适

#### 4\.3\.3 厚重玻璃（Thick Glass）

- **透明度**：90% 不透明度

- **模糊半径**：30dp

- **适用场景**：弹窗、底部导航栏、顶部状态栏

- **视觉感受**：扎实、有分量

### 4\.4 玻璃效果细节

#### 4\.4\.1 高光效果

- 顶部有一道 1dp 高的白色高光（30% 不透明度）

- 高光从左到右有渐变，中间亮两边暗

- 模拟光线从上方照射的效果

#### 4\.4\.2 边框效果

- 1dp 的半透明白色边框

- 边框有渐变，顶部亮底部暗

- 增强玻璃的边缘感

#### 4\.4\.3 内阴影

- 卡片内部有微妙的内阴影

- 顶部和底部各有一道淡淡的阴影

- 增加立体感

#### 4\.4\.4 背景噪点

- 玻璃表面叠加一层极淡的噪点纹理

- 噪点透明度：3\-5%

- 模拟真实玻璃的微小瑕疵，增加真实感

### 4\.5 液态动效

#### 4\.5\.1 按压液态效果

当用户按下玻璃卡片时，产生类似按压液态玻璃的效果：

- 卡片轻微下沉 2dp

- 模糊半径略微减小（模拟被压缩）

- 高光位置略微下移

- 阴影变小变近

- 动画时长：150ms，缓动：EaseOut

#### 4\.5\.2 回弹效果

松开手指后，卡片弹回原位：

- 有轻微的过冲效果（Overshoot）

- 像果冻一样回弹

- 动画时长：300ms，缓动：Overshoot

#### 4\.5\.3 水波纹扩散

点击时从点击位置产生水波纹效果：

- 波纹是半透明的

- 波纹扩散速度适中

- 与玻璃材质融合自然

### 4\.6 不同组件的玻璃效果应用

#### 4\.6\.1 底部导航栏

- 厚重玻璃效果

- 背景重度模糊

- 顶部有高光边框

- 选中项有液态高亮效果

#### 4\.6\.2 顶部状态栏

- 轻薄玻璃效果

- 与内容自然融合

- 滚动时逐渐变得不透明

#### 4\.6\.3 卡片

- 标准玻璃效果

- 悬停 / 按压时有液态反馈

- 不同类型卡片有细微差别

#### 4\.6\.4 按钮

- 主按钮：渐变填充 \+ 玻璃质感

- 次按钮：纯玻璃效果

- 小按钮：轻薄玻璃

#### 4\.6\.5 弹窗

- 厚重玻璃效果

- 背景有半透明遮罩

- 出现时有弹性动画

#### 4\.6\.6 输入框

- 玻璃质感边框

- 聚焦时边框发光

- 输入内容时文字清晰可见

### 4\.7 性能优化

#### 4\.7\.1 模糊性能

- 模糊效果使用硬件加速

- 只在必要时更新模糊内容

- 静态内容模糊缓存

- 低性能设备自动降级

#### 4\.7\.2 降级方案

对于不支持硬件模糊的设备：

- 使用半透明纯色背景替代

- 保留高光和边框效果

- 保证视觉风格一致

- 用户可在设置中手动开关模糊效果

## 5\. 核心功能模块 \- 自定义打卡系统

### 5\.1 打卡系统总览

打卡系统是应用的核心功能。用户可以创建任意类型的打卡项目，每个打卡项目都有独立的设置、数据、统计和分析。

#### 5\.1\.1 打卡类型分类

1. **计数型打卡**：记录次数（如喝水杯数、学习次数）

2. **时长型打卡**：记录时长（如运动时长、学习时长）

3. **数值型打卡**：记录数值（如体重、血压、饮水量 ml）

4. **勾选型打卡**：只需确认完成（如早起、冥想、护肤）

5. **图文型打卡**：可上传图片和文字（如饮食记录、读书笔记）

6. **计时型打卡**：开始 / 结束自动计时（如专注计时）

#### 5\.1\.2 预设打卡模板

应用内置丰富的打卡模板，用户可直接使用或在此基础上修改：

**健康类**：

- 每日饮水

- 饮食记录

- 运动健身

- 睡眠追踪

- 体重记录

- 冥想练习

- 服药提醒

- 排便记录

**学习类**：

- 学习时长

- 阅读打卡

- 单词背诵

- 练字打卡

- 编程练习

- 听力练习

**生活类**：

- 早起打卡

- 早睡打卡

- 护肤打卡

- 家务打卡

- 记账记录

- 喝水提醒

**工作类**：

- 工作时长

- 任务完成

- 番茄钟

- 日报记录

### 5\.2 自定义打卡创建流程

#### 5\.2\.1 创建入口

- 首页右下角浮动按钮（\+）

- 打卡列表页顶部添加按钮

- 模板选择页面

#### 5\.2\.2 创建步骤

采用分步引导式创建，降低用户认知负担：

**第一步：选择类型或模板**

- 顶部展示推荐模板（横向滚动）

- 下方展示打卡类型选择（卡片式）

- 每个类型有图标、名称、简短说明

- 点击类型进入下一步

**第二步：基础设置**

- 打卡名称（输入框，支持 emoji）

- 打卡图标（图标选择器，支持 emoji 和内置图标）

- 打卡颜色（颜色选择器，支持渐变色）

- 打卡单位（如：杯、分钟、次、公斤、ml 等）

- 目标值（每日目标，可选）

- 简短描述（可选）

**第三步：高级设置**

- 打卡时间段（如全天、早上、下午、晚上，或自定义时间段）

- 提醒设置（时间、频率、是否震动）

- 可见性（是否在首页展示）

- 排序位置

- 是否启用 AI 分析

**第四步：预览确认**

- 展示打卡卡片预览效果

- 可返回修改

- 确认创建

#### 5\.2\.3 打卡编辑

- 长按打卡卡片进入编辑模式

- 可修改所有设置

- 修改历史数据不受影响

### 5\.3 饮食打卡详细设计

饮食打卡是最复杂的打卡类型，单独详细设计：

#### 5\.3\.1 饮食记录方式

1. **快速记录**：选择预设食物，一键记录

2. **拍照识别**：AI 识别食物，自动计算热量

3. **手动输入**：手动搜索食物，输入份量

4. **语音输入**：语音描述食物，AI 解析

#### 5\.3\.2 餐次分类

- 早餐

- 上午加餐

- 午餐

- 下午加餐

- 晚餐

- 夜宵

- 自定义餐次

#### 5\.3\.3 食物数据库

- 内置常见食物数据库（离线）

- 包含：热量、蛋白质、碳水、脂肪、膳食纤维等营养信息

- 支持收藏常用食物

#### 5\.3\.4 记录详情

每条饮食记录包含：

- 食物名称

- 份量（克 / 毫升 / 份）

- 热量（大卡）

- 营养成分（蛋白质、碳水、脂肪）

- 用餐时间

- 备注（可选）

- 图片（可选）

- 用餐心情（可选）

#### 5\.3\.5 每日统计

- 今日总热量

- 三大营养素比例饼图

- 各餐次热量分布

- 与目标对比进度条

- AI 饮食建议

### 5\.4 饮水打卡详细设计

#### 5\.4\.1 记录方式

- 一键记录（常用杯型）

- 提醒触发记录

- 自动根据时间建议饮水量

#### 5\.4\.2 杯型设置

- 预设杯型：小杯（200ml）、中杯（300ml）、大杯（500ml）

- 可设置常用杯型，首页快速记录

#### 5\.4\.3 目标设置

- 每日饮水目标（默认 2000ml）

- 智能提醒：根据饮水间隔自动提醒

- 提醒可自定义铃声、震动

#### 5\.4\.5 统计分析

- 每日饮水量趋势

- 饮水时间分布

- 达标率统计

- AI 饮水习惯分析

### 5\.5 运动打卡详细设计

#### 5\.5\.1 运动类型

- 有氧运动：跑步、游泳、骑行、跳绳等

- 力量训练：举重、俯卧撑、深蹲等

#### 5\.5\.3 统计分析

- 每周运动时长

- 运动频率统计

- AI 运动建议

### 5\.6 睡眠打卡详细设计

#### 5\.6\.1 记录方式

## 6\. 时间视图系统

### 6\.1 视图总览

时间视图系统是应用的数据展示核心，支持从分钟到年级别的多维度数据查看。所有视图之间可以无缝切换，并有丝滑的过渡动画。

#### 6\.1\.1 视图层级

```Plain Text
年视图
  └── 月视图
        └── 周视图
              └── 日视图
                    └── 小时视图
                          └── 分钟视图
```

#### 6\.1\.2 切换方式

- 双指缩放：放大进入更细粒度视图，缩小进入更粗粒度视图

- 点击：点击某个时间段进入下一级视图

- 滑动切换：左右滑动切换上一个 / 下一个时间段

- 快速跳转：时间选择器快速跳转到指定日期

### 6\.2 分钟视图

#### 6\.2\.1 适用场景

查看某一小时内的详细打卡记录，精确到分钟。

#### 6\.2\.2 展示方式

- 时间轴：垂直时间轴，每 5 分钟一个刻度

- 记录点：在对应时间位置显示打卡记录

- 记录卡片：点击记录点展开详细信息

- 时间范围：显示 1 小时的内容

#### 6\.2\.3 交互细节

- 上下滑动浏览

- 记录点有脉冲动画，表示活跃

- 展开记录时有流畅的展开动画

- 可左右滑动切换上一小时 / 下一小时

### 6\.3 小时视图

#### 6\.3\.1 适用场景

查看某一天内的打卡时间分布，按小时统计。

#### 6\.3\.2 展示方式

- 24 小时时间轴（横向或纵向）

- 每个小时有柱状图或热力图显示活跃度

- 可切换不同打卡项目查看

- 可叠加多个打卡项目对比

#### 6\.3\.3 数据展示

- 每个小时的打卡次数 / 总时长

- 高峰时段高亮显示

- 空白时段显示为灰色

- 鼠标悬停 / 长按显示详细数据

### 6\.4 日视图

#### 6\.4\.1 适用场景

查看某一天的完整打卡情况，是最常用的视图之一。

#### 6\.4\.2 页面结构

**顶部日期栏**：

- 显示当前日期（如：2024 年 1 月 15 日 星期一）

- 左右箭头切换日期

- 点击弹出日历选择器

**概览卡片**：

- 今日完成率（环形进度图）

- 已完成 / 总打卡数

- 连续打卡天数

- 今日亮点摘要

**时间轴**：

- 垂直时间轴，从早到晚

- 不同类型的打卡用不同颜色标记

- 每条记录有图标和简短文字

- 可展开查看详情

**打卡项目列表**：

- 每个打卡项目今日完成情况

- 进度条显示完成度

- 点击进入该打卡的详情页

#### 6\.4\.3 交互细节

- 左右滑动切换上一天 / 下一天

- 下滑回到今天

- 双指捏合缩小进入周视图

- 点击时间轴上的记录查看详情

### 6\.5 周视图

#### 6\.5\.1 适用场景

查看一周内的打卡趋势和完成情况。

#### 6\.5\.2 展示方式

**日历热力图**：

- 7 天并排展示

- 每天用颜色深浅表示完成度

- 点击某天进入日视图

**趋势图表**：

- 折线图展示本周数据趋势

- 可切换不同打卡项目

- 可对比上周数据

**周统计**：

- 本周总打卡次数

- 平均每日完成率

- 连续达标天数

- 本周最佳 / 最差一天

#### 6\.5\.3 交互细节

- 左右滑动切换上一周 / 下一周

- 点击某一天进入日视图

- 双指放大进入日视图

- 双指缩小进入月视图

### 6\.6 月视图

#### 6\.6\.1 适用场景

查看一个月的打卡概览，发现月度规律。

#### 6\.6\.2 展示方式

**日历视图**：

- 完整的月历布局

- 每天用颜色填充表示完成度（热力图效果）

- 完成的日期有对勾标记

- 今天有特殊高亮

- 点击某天进入日视图

**月度趋势**：

- 折线图 / 柱状图展示月度数据

- 显示平均值、最大值、最小值

**月度统计**：

- 本月总打卡次数

- 本月完成率

- 最长连续打卡天数

#### 6\.6\.3 交互细节

- 左右滑动切换上一月 / 下一月

- 点击某天进入日视图

- 双指放大进入周视图

- 双指缩小进入年视图

#### 6\.7\.1 适用场景

查看全年的打卡情况，看到长期坚持的成果。

#### 6\.7\.2 展示方式

**年度热力图**：

- 类似 GitHub 贡献图的热力图

- 每一小格代表一天

- 颜色深浅表示当天完成度

- 12 个月按行排列

**月度对比**：

- 12 个月的柱状图对比

- 可切换查看不同指标

- 最佳 / 最差月份高亮

**年度总结**：

- 全年总打卡次数

- 全年平均完成率

- 最长连续打卡记录

- 年度关键词 / 总结

- AI 年度分析报告

#### 6\.7\.3 交互细节

- 左右滑动切换上一年 / 下一年

- 点击某月进入月视图

- 点击某天进入日视图

- 双指放大进入月视图

### 6\.8 视图切换动画

#### 6\.8\.1 缩放切换

使用双指缩放切换视图粒度时：

- 内容平滑缩放

- 数据点逐渐聚合或分散

- 缩放过程中保持视觉连续性

- 达到阈值时自动切换到下一级视图

#### 6\.8\.2 点击进入

点击某个时间段进入下一级视图时：

- 被点击的区域放大填充整个屏幕

- 周围内容逐渐淡出

- 新视图的细节逐渐显现

- 动画时长：400ms

## 7\. AI 智能分析系统

### 7\.1 AI 系统总览

AI 智能分析系统是应用的高级功能，通过接入用户自己的 AI 模型 API，为用户提供智能化的数据分析、识别和建议功能。

#### 7\.1\.1 设计原则

1. **可选增强**：AI 功能是可选的，不影响基础功能使用

2. **用户可控**：用户完全控制 AI 的使用，可随时开启 / 关闭

3. **隐私保护**：所有 AI 请求都由用户设备直接发起，数据不经过第三方

4. **成本透明**：清晰展示 Token 消耗，让用户心中有数

5. **离线降级**：没有 AI 时，应用依然完整可用

#### 7\.1\.2 AI 功能分类

1. **多模态识别**：图片识别、语音识别

2. **数据分析**：趋势分析、模式识别、异常检测

3. **智能建议**：个性化建议、目标规划、改进方案

4. **内容生成**：总结报告、打卡文案、鼓励话语

### 7\.2 多模态与非多模态协同架构

这是本应用 AI 系统的核心特色，通过多模态和非多模态模型协同工作，在保证功能强大的同时最大化节省 Token 消耗。

#### 7\.2\.1 工作原理

```Plain Text
用户输入（图片/语音/文字）
      ↓
多模态 AI 模型（如 GPT-4V、Claude 3 Opus 等）
      ↓
输出结构化文字描述
      ↓
非多模态 AI 模型（如 GPT-3.5、本地小模型等）
      ↓
输出最终分析结果/建议
```

#### 7\.2\.2 协同优势

1. **节省成本**：多模态模型通常较贵，只用来做识别；非多模态模型便宜，用来做分析

2. **提高效率**：结构化的文字输入比图片输入 Token 消耗少得多

3. **灵活组合**：用户可以自由搭配不同厂商的模型

4. **降级容易**：多模态模型不可用时，可以手动输入文字描述

#### 7\.2\.3 配置方式

用户可以在 AI 设置中配置：

- 多模态模型：用于图片识别、语音转文字

- 分析模型：用于数据分析、生成建议

- 是否启用协同模式

- Token 消耗预警阈值

#### 7\.2\.4 Token 优化策略

1. **提示词优化**：使用最精简的提示词，减少 Token 消耗

2. **图片压缩**：上传图片前自动压缩到合适尺寸

3. **结果缓存**：相同的分析请求缓存结果，避免重复调用

4. **批量处理**：多条记录合并成一次分析请求

5. **智能截断**：过长的数据自动摘要后再发送

### 7\.3 饮食 AI 分析

#### 7\.3\.1 食物识别

**功能描述**：用户上传食物照片，AI 自动识别食物种类、估算份量、计算热量和营养成分。

**识别流程**：

1. 用户拍照或从相册选择图片

2. 图片压缩处理（降低分辨率以节省 Token）

3. 发送给多模态 AI 模型

4. 多模态模型输出结构化的食物列表（名称、份量、热量、营养成分）

5. 应用解析结果，展示给用户确认

6. 用户确认后保存记录

**识别精度优化**：

- 提示词中要求输出 JSON 格式，便于解析

- 要求估算份量时使用常见参照物（如拳头大小、一碗、一盘等）

- 要求标注识别置信度，低置信度的项目标红提醒用户确认

- 支持用户修正识别结果，修正后的数据用于后续优化提示词

#### 7\.3\.2 营养分析

**功能描述**：分析用户一段时间的饮食数据，给出营养摄入分析和建议。

**分析维度**：

1. **热量分析**：

    - 每日热量摄入趋势

    - 与目标热量对比

    - 各餐次热量分配是否合理

    - 热量波动分析

2. **营养素分析**：

    - 三大营养素（蛋白质、碳水、脂肪）比例

    - 膳食纤维摄入是否充足

    - 维生素和矿物质摄入评估（如果有数据）

    - 营养素摄入均衡性评分

3. **饮食习惯分析**：

    - 用餐时间是否规律

    - 三餐比例是否合理

    - 夜宵频率

    - 食物多样性评估

4. **个性化建议**：

    - 根据用户目标（减脂 / 增肌 / 保持）给出建议

    - 指出饮食中的问题

    - 给出具体的改进建议

    - 推荐更健康的替代食物

#### 7\.3\.3 饮食报告生成

- 周报：每周饮食总结和建议

- 月报：月度饮食分析报告

- 自定义时间段报告

### 7\.4 饮水 AI 分析

#### 7\.4\.1 饮水习惯分析

**分析维度**：

1. **饮水量分析**：

    - 每日饮水量是否达标

    - 饮水量波动情况

    - 与推荐饮水量对比

2. **饮水时间分析**：

    - 饮水时间分布

    - 是否有长时间不喝水的情况

    - 饭前饭后饮水习惯

    - 睡前饮水情况

3. **饮水规律分析**：

    - 平均饮水间隔

    - 最活跃饮水时段

    - 容易忘记喝水的时段

#### 7\.4\.2 智能提醒优化

AI 根据用户的饮水习惯，优化提醒时间：

- 识别用户通常什么时候容易忘记喝水

- 在合适的时间提前提醒

- 调整提醒频率，避免过多打扰

- 根据天气、运动量动态调整每日饮水目标

#### 7\.4\.3 饮水建议

- 根据用户的体重、年龄、性别、活动量计算推荐饮水量

- 给出改善饮水习惯的具体建议

- 提醒用户不要一次性喝太多水

- 建议饭前喝水的好处等健康知识

### 7\.5 运动 AI 分析

#### 7\.5\.1 运动效果分析

- 运动频率是否足够

- 运动强度是否合适

- 热量消耗分析

- 运动类型多样性评估

#### 7\.5\.2 运动计划建议

### 7\.7 习惯养成 AI 分析

## 9\. 动画与交互设计规范

### 9\.1 动画设计原则

#### 9\.1\.1 物理真实感

所有动画都要模拟真实世界的物理规律：

- 加速和减速符合惯性

- 弹性动作用弹簧曲线

- 物体运动有重量感

- 避免机械的线性运动

#### 9\.1\.2 响应迅速

- 用户操作后 16ms 内要有视觉反馈

- 动画时长适中，不拖沓也不突兀

- 优先保证跟手性，再追求完美

#### 9\.1\.3 层次分明

- 不同层级的元素动画速度不同

- 前景元素动得快，背景元素动得慢

- 创造视差效果，增强空间感

#### 9\.1\.4 克制优雅

- 动画不要过于花哨炫技

- 每个动画都要有明确的目的

- 重点内容突出，次要内容弱化

- 整体风格统一协调

### 9\.2 动画时长与缓动曲线

#### 9\.2\.1 标准时长

- 快速反馈（按钮点击、开关切换）：100\-150ms

- 标准过渡（页面切换、元素展开）：250\-350ms

- 复杂动画（弹窗出现、列表加载）：400\-500ms

- 装饰性动画（粒子效果、背景动画）：1\-3 秒循环

#### 9\.2\.2 缓动曲线

- **标准缓动（Standard Ease）**：`EaseInOutCubic`

    - 用途：大多数标准动画

    - 特点：慢入慢出，中间快，自然流畅

- **减速缓动（Decelerate Ease）**：`EaseOutCubic`

    - 用途：元素进入屏幕

    - 特点：快速开始，慢慢停止

- **加速缓动（Accelerate Ease）**：`EaseInCubic`

    - 用途：元素退出屏幕

    - 特点：慢慢开始，快速结束

- **弹性缓动（Spring）**：自定义弹簧曲线

    - 用途：按钮回弹、卡片弹出、下拉刷新

    - 特点：有过冲和回弹，像弹簧一样

- **线性（Linear）**：匀速运动

    - 用途：旋转、进度条等

    - 特点：速度恒定

### 9\.3 页面切换动画

#### 9\.3\.1 底部导航切换

- 点击底部导航项切换页面时

- 当前页面淡出并轻微缩小

- 新页面淡入并轻微放大

- 底部导航项有液态高亮移动效果

- 动画时长：300ms

#### 9\.3\.2 详情页进入

- 卡片中的元素自然过渡到详情页对应位置

- 背景逐渐模糊

- 动画时长：400ms

#### 9\.3\.3 详情页返回

- 返回时详情页缩小回到原卡片位置

- 与进入动画对称但稍快一些

- 动画时长：350ms

- 如果用户在详情页滚动过，先平滑滚动到顶部再开始返回动画

#### 9\.3\.4 全屏弹窗

### 10\.1 数据存储架构

- type: 打卡类型

- unit: 单位

- id: 主键

- value: 记录数值

- date: 记录日期

- time: 记录时间

- note: 备注

- images: JSON 图片路径列表

- id: 主键

- id: 主键

- model: 模型名称

### 10\.4 隐私保护

#### 10\.4\.1 数据本地化

- 所有用户数据 100% 存储在本地

- 不上传到任何服务器

- 不收集任何用户信息

- 没有账号系统

#### 10\.4\.2 网络使用透明

- 只有 AI 功能会使用网络

- 所有网络请求都是用户主动触发的

- 可在设置中查看网络请求日志

- 可完全禁用网络权限

#### 10\.4\.3 API Key 安全

- API Key 使用系统密钥库加密存储

- 不会明文显示

- 不会上传到任何地方

- 应用卸载时自动删除

#### 10\.4\.4 图片隐私

- 打卡图片存储在应用私有目录

- 其他应用无法访问

- 用户可选择是否保存到系统相册

### 10\.5 数据安全

#### 10\.5\.1 数据库加密

- 可选启用数据库加密（SQLCipher）

- 用户设置密码

- 启用后所有数据都加密存储

- 忘记密码无法恢复（提醒用户牢记密码）

#### 10\.5\.2 应用锁

- 支持应用锁功能

- 解锁方式：密码、指纹、面部识别

- 可设置自动锁定时间

- 保护用户隐私数据不被他人查看

#### 10\.5\.3 敏感内容保护

- 最近任务列表中隐藏应用内容

- 支持截图禁用（可选）

- 敏感数据默认模糊显示，点击才显示详情

### 10\.6 数据删除

#### 10\.6\.1 单条删除

- 可删除单条打卡记录

- 删除时有确认提示

- 删除后可撤销（5 秒内）

#### 10\.6\.2 批量删除

- 可按时间范围批量删除

- 可按打卡项目删除

- 删除前需要确认

- 不可撤销（提醒用户）

#### 10\.6\.3 清空所有数据

- 一键清空所有数据

- 有多重确认

- 清空后应用恢复初始状态

- 不可恢复
